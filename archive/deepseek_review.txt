#### **2. Equivalency Table Caching Strategy**
**Keep on-demand with smart indexing.** Here's why:

```sql
-- Current Indexing (Good):
CREATE INDEX idx_equivalency_members_card_id ON equivalency_members(card_id);

-- Recommended Enhancement:
CREATE INDEX idx_equivalency_members_with_group 
ON equivalency_members(card_id, group_id) 
INCLUDE (confidence);
```

**Rationale:**
- Equivalency lookups are **rare** (<5% of cards in Standard format)
- Table grows slowly (mainly functional reprints)
- Index-only scan with `INCLUDE` eliminates table access entirely
- Memory saved for primary card cache is more valuable

#### **3. Name Mismatch Resolution**
**Three-layer approach recommended:**

```python
def resolve_name_mismatch(parsed_name: str, db_name: str) -> Status:
    # Layer 1: Exact normalization
    normalized_parsed = normalize_name(parsed_name)  # Lowercase, strip hyphens
    normalized_db = normalize_name(db_name)
    
    if normalized_parsed == normalized_db:
        return Status.VALID
    
    # Layer 2: Character distance (Levenshtein)
    if levenshtein_distance(normalized_parsed, normalized_db) <= 2:
        return Status.WARNING
    
    # Layer 3: Known patterns lookup (cached)
    known_patterns = {  # Pre-populated cache
        "mew ex": "mew-ex",
        "boss's orders": "boss’s orders",
    }
    if parsed_name.lower() in known_patterns:
        return Status.WARNING
    
    return Status.INVALID
```

**Implementation Note:** Build the known patterns cache from historical validation logs.
