REASONING:
We are implementing a hybrid parsing system for the "Magic Paste" feature. The user pastes a string (e.g., a team list from a PokÃ©mon forum) and we want to extract structured data (pokemon, items, moves, etc.). We use regex first for speed and reliability, and if that fails or is incomplete, we fall back to an LLM.

## Data Flow Overview

1. **User Interaction**: The user pastes text into a textarea or contenteditable div in a React component.
2. **Parse Request**: The component sends the raw string to a parsing service (which implements the hybrid approach).
3. **Hybrid Parsing**:
   a. **Regex Parsing**: First, attempt to parse with regex. This should be fast and provide structured data if the format is known.
   b. **LLM Fallback**: If regex parsing fails (or returns incomplete data), call an LLM (like GPT-4) to parse the text. We must be cautious about latency and cost.
4. **Validation**: The parsed data (from regex or LLM) is then validated with Zod. We want to handle partial data gracefully.
5. **State Update**: The validated data (or errors) is returned to the React component to update the UI.

## Architecture Considerations

We'll structure the parsing service as a separate module (or API route in Next.js) to keep the frontend lightweight. However, since regex parsing is fast, we could do it on the client. But LLM calls require an API key and are done server-side.

We can design two flows:

### Client-side Regex, Server-side LLM
- Do regex parsing in the browser. If it fails, send the raw string to a serverless function (Next.js API route) that calls the LLM and returns the structured data.

### Server-side Hybrid
- Send the raw string to an API route that does both regex and LLM (if needed). This keeps the client simple and allows us to hide the LLM API key.

We'll choose the server-side hybrid for simplicity and security.

```typescript
  async parse(rawText: string): ParseResult {
    // Phase 1: Regex Parsing (client or server-side)
    const regexResult = await regexParser.extract(rawText);
    
    if (regexResult.confidence > 0.95) {
      return {
        data: regexResult.data,
        source: 'regex',
        confidence: regexResult.confidence,
        warnings: regexResult.warnings

    // Phase 2: LLM Enhancement
    const llmResult = await llmParser.enhance(
      rawText, 
      regexResult.partialData, // Pass partial regex results
      regexResult.confidenceThresholds

    return {
      data: mergeResults(regexResult, llmResult),
      source: 'hybrid',
      confidence: calculateCombinedConfidence(regexResult, llmResult),
```

### 4. Validation Pipeline with Error Recovery

```typescript
// lib/validation/progressive-validator.ts
class ProgressiveValidator {
  async validate(
    data: unknown, 
    level: ValidationLevel = 'partial'
  ): ValidationResult {
    
    const result = await PartialTeamSchema.safeParseAsync(data);
    
    if (!result.success) {
      // Don't fail entirely - extract what we can
      return await this.handlePartialErrors(result.error, data);
    }
    
    // Apply level-specific validations
    const enhancedResult = await this.applyLevelValidations(
      result.data, 
      level
    );
    
    return enhancedResult;
  }
  
  private async handlePartialErrors(
    error: ZodError, 
    originalData: unknown
  ): Promise<ValidationResult> {
    // Extract recoverable data from partial parse
    const recoveredData = this.extractRecoverableData(
      error, 
      originalData
    );
    
    // Categorize errors by severity
    const categorizedErrors = this.categorizeErrors(error);
    
    return {
      success: false,
      data: recoveredData,
      errors: categorizedErrors,
      warnings: this.generateWarnings(categorizedErrors),
      suggestions: this.generateSuggestions(categorizedErrors, originalData),
    };
  }
  
  private extractRecoverableData(
    error: ZodError, 
    data: unknown
  ): Partial<TeamData> {
    // Traverse error tree to find valid partial data
    // Use Zod's path information to extract what passed validation
    const validPaths = this.getValidPaths(error);
    return extractValidSubset(data, validPaths);
  }
}
```

### 9. Key Architectural Decisions

1. **Progressive Disclosure**: Show minimal feedback initially, expand details on demand
2. **Recoverable Validation**: Never fail completely - always extract what we can
3. **Confidence-based UI**: Visual indicators for inferred/low-confidence data
4. **Quick Fix Integration**: One-click corrections for common issues
5. **State Preservation**: Maintain original text alongside parsed data
6. **Multi-pass Validation**: Allow re-validation after manual corrections

### 10. Performance Considerations

- Cache regex parsing results
- Implement request deduplication
- Use optimistic updates for quick fixes
- Lazy-load LLM parser only when needed
- Implement progressive loading in UI (skeleton â partial â complete)

This architecture provides immediate feedback while gracefully handling the ambiguity inherent in natural language parsing, giving users clear paths to correct issues while preserving their work.

### 5. Complete API Route with Validation

```typescript
// app/api/parse/resume/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { z } from 'zod'
import { resumeParserRatelimit } from '@/lib/rate-limit'
import { parseWithRegex } from '@/lib/regex-parser'
import { LLMParserService } from '@/lib/llm-service'

// Zod schema for validation (shared with frontend)
const ResumeSchema = z.object({
  name: z.string(),
  email: z.string().email(),
  phone: z.string(),
  summary: z.string(),
  experience: z.array(z.object({
    title: z.string(),
    company: z.string(),
    startDate: z.string(),
    endDate: z.string()
  })),
  education: z.array(z.object({
    degree: z.string(),
    school: z.string(),
    graduationYear: z.number()
  }))
})

const llmService = new LLMParserService()

export async function POST(request: NextRequest) {
  try {
    // 1. Rate limiting
    const identifier = request.ip ?? 'anonymous'
    const { success } = await resumeParserRatelimit.limit(identifier)
    if (!success) return rateLimitedResponse()
    
    // 2. Get and validate input
    const { text } = await request.json()
    if (!text || typeof text !== 'string' || text.length > 10000) {
      return NextResponse.json(
        { error: 'Invalid input' },
        { status: 400 }
      )
    }
    
    // 3. Attempt regex parsing first
    const regexResult = parseWithRegex(text)
    if (regexResult.success) {
      return NextResponse.json({
        data: regexResult.data,
        method: 'regex',
        cached: false
      })
    }
    
    // 4. Check cache for similar inputs
    const cacheKey = `resume_parse:${hashString(text)}`
    const cached = await getFromCache(cacheKey)
    if (cached) {
      return NextResponse.json({
        data: cached,
        method: 'llm',
        cached: true
      })
    }
    
    // 5. LLM fallback with timeout
    const controller = new AbortController()
    const timeout = setTimeout(() => controller.abort(), 15000)
    
    try {
      const llmResult = await llmService.parseResume(text)
      
      // 6. Validate against Zod schema
      const validation = ResumeSchema.safeParse(llmResult)
      if (!validation.success) {
        throw new Error('LLM output validation failed')
      }
      
      // 7. Cache successful result (24 hours)
      await setCache(cacheKey, validation.data, 86400)
      
      return NextResponse.json({
        data: validation.data,
        method: 'llm',
        cached: false
      })
      
    } catch (error) {
      if (error.name === 'AbortError') {
        return NextResponse.json(
          { error: 'Processing timeout' },
          { status: 408 }
        )
      }
      throw error
    } finally {
      clearTimeout(timeout)
    }
    
  } catch (error) {
    console.error('Resume parsing error:', error)
    return NextResponse.json(
      { error: 'Failed to parse resume' },
      { status: 500 }
    )
  }
}
```

```tsx
import React, { createContext, useContext, useReducer, ReactNode } from 'react';
import { MagicPasteStateSchema } from './schemas';

type MagicPasteState = z.infer<typeof MagicPasteStateSchema>;

type Action =
  | { type: 'OPEN_MODAL'; payload: string }
  | { type: 'CLOSE_MODAL' }
  | { type: 'SET_MODE'; payload: 'plain' | 'enhanced' }
  | { type: 'SET_ENHANCED_CONTENT'; payload: string }
  | { type: 'SET_LOADING'; payload: boolean }
  | { type: 'SET_ERROR'; payload: string };

const initialState: MagicPasteState = {
  isOpen: false,
  rawContent: '',
  enhancedContent: '',
  selectedMode: 'plain',
  isLoading: false,
  error: '',
};

const MagicPasteContext = createContext<{
  state: MagicPasteState;
  dispatch: React.Dispatch<Action>;
} | undefined>(undefined);

const reducer = (state: MagicPasteState, action: Action): MagicPasteState => {
  switch (action.type) {
    case 'OPEN_MODAL':
      return { ...state, isOpen: true, rawContent: action.payload };
    case 'CLOSE_MODAL':
      return initialState;
    case 'SET_MODE':
      return { ...state, selectedMode: action.payload };
    case 'SET_ENHANCED_CONTENT':
      return { ...state, enhancedContent: action.payload };
    case 'SET_LOADING':
      return { ...state, isLoading: action.payload };
    case 'SET_ERROR':
      return { ...state, error: action.payload };
    default:
      return state;
  }
};

export const MagicPasteProvider = ({ children }: { children: ReactNode }) => {
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <MagicPasteContext.Provider value={{ state, dispatch }}>
      {children}
    </MagicPasteContext.Provider>
  );
};

export const useMagicPaste = () => {
  const context = useContext(MagicPasteContext);
  if (context === undefined) {
    throw new Error('useMagicPaste must be used within a MagicPasteProvider');
  }
  return context;
};
```

```typescript
// 5.1 Create clean text processor
class PlainTextProcessor {
  process(content: RawPasteContent): ProcessedContent {
    let cleaned = content.rawText;
    
    // Remove excessive whitespace
    cleaned = cleaned.replace(/\s+/g, ' ');
    
    // Normalize line endings
    cleaned = cleaned.replace(/\r\n/g, '\n');
    
    // Remove hidden characters
    cleaned = cleaned.replace(/[\u200B-\u200D\uFEFF]/g, '');
    
    // Trim each line
    cleaned = cleaned.split('\n').map(line => line.trim()).join('\n');
    
    return {
      cleanedText: cleaned.trim(),
      structuredData: { headings: [], tables: [], codeBlocks: [] },
      tags: ['plain-text'],
      confidenceScore: 1.0,
    };
  }
}
```

```typescript
class EnhancementService {
  private readonly openAIClient: OpenAI;
  private readonly cache: Map<string, EnhancedContent>;
  
  async enhance(content: RawPasteContent): Promise<EnhancedContent> {
    const cacheKey = this.generateCacheKey(content);
    if (this.cache.has(cacheKey)) {
      return this.cache.get(cacheKey)!;
    }
    
    const prompt = this.buildEnhancementPrompt(content);
    const response = await this.openAIClient.chat.completions.create({
      model: "gpt-4",
      messages: [
        {
          role: "system",
          content: "You are a document enhancement assistant. Clean, structure, and enhance pasted content."
        },
        { role: "user", content: prompt }
      ],
      temperature: 0.3,
    });
    
    const enhanced = this.parseLLMResponse(response.choices[0].message.content);
    this.cache.set(cacheKey, enhanced);
    
    return EnhancedContentSchema.parse(enhanced);
  }
}
```

```typescript
        this.insertContent(state);
      }
    });
  }

    const content = state.selectedMode === 'plain'
      ? state.plainContent?.cleanedText
      : state.enhancedContent?.cleanedText;
    
    if (content) {
      this.editor.insertText(content, {
        format: state.processingOptions.formatting,
      });
      
      // Add structured data if available
      if (state.enhancedContent?.structuredData) {
        this.applyStructure(state.enhancedContent.structuredData);
      }
    }
  }
```

```typescript
// 11.1 Implement caching and debouncing
class OptimizedEnhancementService extends EnhancementService {
  private readonly debouncedRequests = new Map<string, Promise<EnhancedContent>>();
  
  async enhanceWithDebounce(
    content: RawPasteContent,
    delay: number = 500
  ): Promise<EnhancedContent> {
    const cacheKey = this.generateCacheKey(content);
    
    // Cancel previous request for same content
    if (this.debouncedRequests.has(cacheKey)) {
      // Optionally cancel previous request
    }
    
    return new Promise((resolve) => {
      const timeoutId = setTimeout(async () => {
        const result = await super.enhance(content);
        this.debouncedRequests.delete(cacheKey);
        resolve(result);
      }, delay);
      
      // Store timeout for potential cancellation
      this.debouncedRequests.set(cacheKey, {
        then: (onFulfilled) => {
          clearTimeout(timeoutId);
          return Promise.resolve(onFulfilled);
        },
      } as any);
    });
  }
}
```

#### **Step 12: Error Handling & Fallbacks**
```typescript
// 12.1 Comprehensive error handling
class ResilientMagicPaste {
  async processWithFallback(content: RawPasteContent) {
    try {
      // Try enhanced processing
      const enhanced = await enhancementService.enhance(content);
      return { mode: 'enhanced', content: enhanced };
    } catch (error) {
      console.warn('Enhanced processing failed, falling back to plain:', error);
      
      // Fallback 1: Use cached enhancement if available
      const cached = enhancementCache.get(content);
      if (cached) return { mode: 'enhanced', content: cached };
      
      // Fallback 2: Use plain processing
      const plain = plainTextProcessor.process(content);
      return { mode: 'plain', content: plain };
      
      // Fallback 3: Return raw with warning
      return {
        mode: 'raw',
        content: { cleanedText: content.rawText, tags: ['fallback'] },
        warning: 'Processing unavailable, using raw text',
      };
    }
  }
}
```
